# Исправление 403 ошибок корзины для неавторизованных пользователей

## Проблема
При запуске приложения неавторизованным пользователем возникали 403 ошибки:
```
Error in GET /api/cart?_t=1758616382175: Request failed with status code 403
Ошибка при загрузке корзины: {"code": 403, "errors": [], "message": "Доступ запрещен. Требуется роль: CLIENT", "status": "error"}
```

## Корень проблемы
1. **Серверная логика**: сервер требует роль CLIENT для доступа к корзине
2. **Клиентская логика**: приложение пыталось загружать корзину для всех пользователей, включая неавторизованных
3. **Безопасность**: отсутствовала проверка роли пользователя перед запросами к API

## Решение

### 1. Исправление fetchCart thunk
Добавлена проверка авторизации перед запросом к серверу:

```javascript
// В mobile/src/entities/cart/model/slice.js
export const fetchCart = createAsyncThunk(
    'cart/fetchCart',
    async (forceRefresh = false, { rejectWithValue, getState }) => {
        const state = getState();
        const isAuthenticated = state.auth?.user?.id;
        
        // Для неавторизованных пользователей НЕ делаем запрос к серверу
        if (!isAuthenticated) {
            return { 
                data: {
                    items: state.cart?.items || [],
                    summary: { /* локальная статистика */ }
                }, 
                fromCache: true 
            };
        }
        
        // Для авторизованных - делаем запрос к серверу
        const response = await CartService.getCart();
        // ...
    }
);
```

### 2. Исправление fetchCartStats
Аналогичная логика для статистики корзины:

```javascript
export const fetchCartStats = createAsyncThunk(
    'cart/fetchCartStats',
    async (_, { rejectWithValue, getState }) => {
        const state = getState();
        const isAuthenticated = state.auth?.user?.id;
        
        // Для неавторизованных возвращаем локальную статистику
        if (!isAuthenticated) {
            return {
                totalBoxes: state.cart?.totalBoxes || 0,
                totalItems: state.cart?.totalItems || 0,
                // ...
            };
        }
        
        // Для авторизованных - запрос к серверу
        const response = await CartService.getCartStats();
        // ...
    }
);
```

### 3. Улучшение middleware
Обновлена обработка ошибок авторизации:

```javascript
// В mobile/src/entities/cart/model/middleware.js
if (action.type.startsWith('cart/') && action.type.endsWith('/rejected')) {
    const errorMessage = action.payload || 'Произошла ошибка';
    
    const isAuthError = errorMessage.includes('Доступ запрещен') ||
                       errorMessage.includes('403') ||
                       errorMessage.includes('Корзина недоступна');
    
    if (isAuthError) {
        // Только логируем в DEV, не показываем пользователю
        if (__DEV__) {
            console.log(`🔒 CartReloadMiddleware: Auth/Role error (expected):`, errorMessage);
        }
    } else {
        // Показываем только реальные ошибки
        store.dispatch(addNotification({
            // ...
        }));
    }
}
```

### 4. Оптимизация автозагрузки
Обновлена логика в MainScreen:

```javascript
// В mobile/src/screens/main/ui/MainScreen.jsx
if (isCartAvailable) {
    // Для авторизованных пользователей делаем форсированное обновление
    // Для неавторизованных - просто проверяем локальное состояние
    dispatch(fetchCart(user?.role === 'CLIENT'));
}
```

## Результат исправлений

### ✅ Преимущества
1. **Устранение 403 ошибок**: неавторизованные пользователи больше не получают ошибки доступа
2. **Безопасность**: запросы к серверу отправляются только для авторизованных CLIENT'ов
3. **Оптимизация**: снижение нагрузки на сервер за счет предотвращения ненужных запросов
4. **UX**: отсутствие ошибочных уведомлений для неавторизованных пользователей
5. **Производительность**: локальная обработка гостевой корзины

### 🔒 Безопасность
- **Контроль доступа**: проверка роли пользователя на клиенте
- **Серверная защита**: сохранена серверная валидация доступа
- **Изоляция данных**: каждая роль видит только доступные ей данные

### 📱 Поведение по ролям

#### Неавторизованные пользователи (Гости)
- ✅ Локальная корзина работает
- ✅ Добавление товаров в AsyncStorage
- ✅ Просмотр локальной статистики
- ❌ Нет запросов к серверной корзине

#### CLIENT роль
- ✅ Полный доступ к корзине
- ✅ Синхронизация с сервером
- ✅ Автоматическое объединение гостевой корзины
- ✅ Все функции корзины

#### ADMIN, EMPLOYEE, SUPPLIER, DRIVER роли
- ❌ Корзина недоступна (скрыта в UI)
- ❌ Нет запросов к API корзины
- ✅ Отсутствие ошибок 403

## Технические детали

### Проверка доступности корзины
```javascript
const isCartAvailable = userRole === 'CLIENT' || !isAuthenticated;
```

### Логика thunk'ов
1. **Проверка роли** → определение доступности
2. **Для неавторизованных** → возврат локальных данных
3. **Для CLIENT** → запрос к серверу
4. **Для других ролей** → rejectWithValue

### Обработка ошибок
- **403 ошибки авторизации**: скрываются от пользователя, логируются в DEV
- **Реальные ошибки**: отображаются через уведомления
- **Валидация на уровне middleware**: дополнительная защита

## Миграция и совместимость

✅ **Полная обратная совместимость**:
- Существующие функции корзины работают без изменений
- Гостевая корзина продолжает функционировать
- Авторизованные клиенты получают полный функционал

✅ **Безопасный запуск**:
- Никаких ошибок при первом запуске приложения
- Корректная работа для всех типов пользователей
- Плавная авторизация с объединением корзин

Это решение полностью устраняет 403 ошибки при сохранении безопасности и функциональности приложения.
